# Design System: Учитель Русского Языка

## 1. Overview

**Creative North Star: "Голос Учителя"**

Система рождается из физического объекта: школьная тетрадь в линейку — бумага, тронутая временем, синие горизонтальные полосы через каждые 34px, красная вертикальная поля слева. Не как декорация, а как рабочая среда. Ученик видит знакомое и начинает работать без объяснений.

Голос учителя встроен в детали: рукописный заголовок на Caveat, моноширинные лейблы на Courier Prime для номеров и подписей как разметка на полях, засечный PT Serif для условий — как язык хорошего учебника. Лёгкий наклон кнопки в `-0.4°` — не баг, а подпись. Оценочная «печать» повёрнута на `-8°` — как штамп от руки. Каждая деталь зарабатывает своё место.

Система отказывается от всего, что опошляет образование: корпоративных синих градиентов ЕГЭ-платформ, мультяшных иконок и геймификационных эффектов, советской канцелярской тяжести. Вместо этого — точность без холода, профессионализм без дистанции.

**Key Characteristics:**
- Тетрадная текстура фона как несущая конструкция, а не декор
- Рукописный шрифт в заголовках и кнопках как подпись учителя
- Плоская иерархия глубины: карточки через прозрачность, не тени
- Красный — только для поля, ошибок и текущего состояния; нигде кроме
- Минимальный наклон кнопок работает именно от своей скупости

## 2. Colors: Тетрадная Палитра

Каждый цвет — роль из физической тетради. Никакого цвета «для красоты».

### Primary
- **Кремовый фон** (#f3f1e8): фон страницы. Это бумага — тёплая, слегка состаренная. Используется как `background-color` тела и базовый слой всех поверхностей. Повторяется в `repeating-linear-gradient` с `--rule` для создания линеек.
- **Тёмно-синий текст** (#233047): основной текст и фон кнопок. Почти чёрный, но синей тональности — как тушь. Всё, что нужно читать, написано этим цветом.

### Secondary
- **Красный акцент** (#b5392f): интерактивные акценты, ошибки, активный вопрос (`.active`), номера вопросов (`.qnum`). Используется скупо — только там, где нужно остановить взгляд.
- **Тёмный красный** (#8c2a22): hover-состояние кнопок исключительно. Никогда в тексте, никогда в статичных элементах.

### Tertiary
- **Зелёный успех** (#3f7a4f): правильный ответ, выполненный вопрос (`.done`). Сдержанный лесной зелёный, не сигнальный.
- **Янтарный частичный** (#b9912f): частично верный ответ, пропущенная запятая (`.missed`), hover кнопки «Показать образец». Не предупреждение — уточнение.
- **Синий информационный** (#2c5f8a): выделенные слова в условии задания (`.qtext i`). Только внутри карточек вопросов.

### Neutral
- **Голубые линейки** (#c9d6e3): линейки фона, границы карточек, пунктирные разделители в `match` и `transform`. Это сетка тетради — повсюду, но не заметно.
- **Розовая поля** (#e2a3a0): вертикальная линия `.margin-line` слева. Единственный декоративный элемент, несущий прямую семантику: как красная поля в настоящей тетради.
- **Приглушённый текст** (#4b5a73): вторичный текст, подзаголовки `.subtitle`, подсказки `.qhint`, неактивные лейблы. Тот же тёмно-синий, но тише.
- **Карандашный серый** (#7c7567): подписи, footer, нейтральные лейблы по умолчанию. Цвет карандашной пометки — не важно, но присутствует.

### Named Rules
**Правило Красного.** `--red` (#b5392f) — не цвет бренда, а цвет внимания. Он появляется только там, где нужна реакция: ошибка, текущее задание, номер вопроса, активный кружок прогресса. На декоративных элементах запрещён.

**Правило Прозрачности.** Карточки никогда не белые. Они `rgba(255,255,255,0.55)` — поверхность над бумагой, а не отдельный лист. Чистый белый разрушает тетрадную иллюзию.

## 3. Typography: Три Голоса

**Display Font:** Caveat, 700 (fallback: cursive)
**Body Font:** PT Serif (fallback: serif)
**Label Font:** Courier Prime (fallback: monospace)

**Характер:** Рукопись встречается с учебником. Caveat — голос учителя: живой, с наклоном, узнаваемый. PT Serif — язык учебника: точный, читаемый, без претензий. Courier Prime — разметка: нейтральная и честная, как номера на полях.

Все три шрифта подключаются через Google Fonts единым `<link>`:
```
https://fonts.googleapis.com/css2?family=Caveat:wght@600;700&family=PT+Serif:ital,wght@0,400;0,700;1,400&family=Courier+Prime:wght@400;700&display=swap
```

### Hierarchy
- **Display** (Caveat, 700, 40px, line-height 1.05): заголовок `h1` на каждой странице. Единственный рукописный элемент в крупном масштабе. Не использовать мельче 32px — теряет характер.
- **Button** (Caveat, 700, 19px, rotate −0.4deg): кнопки `.btn`. Рукописный в действии — единственный случай наклона шрифта.
- **Body** (PT Serif, 400, 18px, line-height 1.45): условия вопросов `.qtext`. Основной читаемый текст. Курсив через `<i>` — цвет `--blue` (#2c5f8a) для выделений.
- **Body Small** (PT Serif, italic, 15px): подзаголовки `.subtitle`. Тот же регистр, меньший масштаб, курсив.
- **Option** (PT Serif, 400, 16px): варианты ответов `.opt`. Читаемый масштаб для выбора.
- **Label** (Courier Prime, 400, 12px, letter-spacing 0.12em, uppercase): `.eyebrow`, `.qnum`, `footer`. Технический уровень — для навигации, не для чтения.
- **Mono Accent** (Courier Prime, 400, 12.5–13px): лейблы вариантов сопоставления `.mbtn`, кнопки самооценки `.gbtn`, кнопка «Показать образец». Метаданные об ответе.

### Named Rules
**Правило Трёх.** В системе ровно три шрифтовые семьи с нераздельными ролями. Четвёртый шрифт запрещён — ни для лендинга, ни для новых заданий.

**Правило Наклона.** Только кнопки `.btn` имеют `transform: rotate(-0.4deg)`. Ни `h1`, ни карточки, ни другие элементы. Эффект работает именно от своей единственности.

## 4. Elevation

Система принципиально плоская. Глубина передаётся через прозрачность и границы, а не через тени.

Карточки `.card` используют `background: rgba(255,255,255,0.55)` — они парят над тетрадным фоном за счёт прозрачности, не подъёма. Блоки пунктуации `.ptext` обозначены `border: 1px dashed var(--rule)` — штрихпунктир отделяет рабочую зону без имитации глубины. Варианты ответов `.opt` используют `border: 1.5px solid var(--rule)` в состоянии покоя, переходя к `border-color: var(--green)` или `var(--red)` после ответа — изменение цвета границы вместо подъёма.

Единственное исключение: `.clip.active` имеет `box-shadow: 0 0 0 3px rgba(181,57,47,0.12)`. Это не тень глубины, а световое кольцо — маркер текущего состояния.

### Named Rules
**Правило Плоскости.** `box-shadow` на карточках, кнопках и контейнерах запрещён. Тень появляется только как маркер состояния (`.clip.active`), никогда как знак глубины или иерархии.

## 5. Components

### Кнопки
Самый характерный компонент системы — рукотворность в действии.

- **Форма:** радиус 3px — почти прямые углы, как обрезанный край листа
- **Primary (`.btn`):** фон `--ink`, текст `--paper`, Caveat 700 19px, padding 7px 20px 9px, `transform: rotate(-0.4deg)`. Hover: фон `--red-dark`. Transition 0.15s.
- **Ghost (`.btn.ghost`):** прозрачный фон, цвет `--ink`, граница 1.5px solid `--ink`. Используется как вторичное действие рядом с primary.
- **Disabled:** `opacity: 0.35`, `cursor: default`. Цвет не меняется.
- **`.nav` с одной кнопкой:** добавить `style="justify-content:flex-end;"` на контейнер `.nav` напрямую.

### Карточка вопроса (`.card`)
- **Углы:** 4px
- **Фон:** rgba(255,255,255,0.55) — полупрозрачный над тетрадным фоном
- **Граница:** 1px solid `--rule`
- **Padding:** 22px 22px 18px
- **Порядок вложения:** `.qnum` → `.qtext` → тело вопроса → `.feedback` → `.nav`. Строго этот порядок.

### Варианты MCQ (`.opt`)
- **Default:** граница 1.5px solid `--rule`, прозрачный фон, цвет `--ink`, ширина 100%
- **Hover:** `border-color: --ink-soft`
- **Correct:** граница `--green`, фон rgba(63,122,79,0.08), псевдоэлемент `::after` «✓» на Caveat 24px 700, цвет `--green`
- **Wrong:** граница `--red`, фон rgba(181,57,47,0.07), псевдоэлемент `::after` «✗» на Caveat 24px 700, цвет `--red`
- После ответа все кнопки блокируются (`disabled`)

### Индикатор прогресса (`.clip`)
Ряд кружков 22×22px, gap 6px.

- **Default:** граница 2px solid `--rule`, текст `--pencil`, фон rgba(255,255,255,0.4)
- **Done (`.done`):** граница и текст `--green`, фон rgba(63,122,79,0.08) — **только при `earned[i] > 0`**
- **Miss (`.miss`):** граница и текст `--red`, фон rgba(181,57,47,0.08) — при `earned[i] = 0`
- **Active (`.active`):** граница и текст `--red`, `box-shadow: 0 0 0 3px rgba(181,57,47,0.12)`

### Обратная связь (`.feedback`)
По умолчанию `display: none`. Появляется через добавление класса состояния.

- **Ok:** фон rgba(63,122,79,0.08), цвет `--green`
- **Bad:** фон rgba(181,57,47,0.07), цвет `--red-dark`
- **Mid:** фон rgba(185,145,47,0.08), цвет `--red-dark`

⚠ **Известное отклонение:** в существующем коде feedback использует `border-left: 3px solid`. Это нарушает абсолютный запрет системы на боковые полосы. При следующей правке заменить на `border: 1px solid` по всему периметру или убрать границу полностью.

### Печать результата (`.stamp`)
- Круг 130×130px, граница 4px solid `--red`, цвет `--red`
- Caveat 700 64px, `transform: rotate(-8deg)`, `margin: 10px auto 18px`
- `::before`: пунктирный круг `inset: -10px`, `opacity: 0.5`
- Оценки: ≥90% → 5, ≥75% → 4, ≥55% → 3, <55% → 2

### Навигация (`.nav`)
- `display: flex`, `justify-content: space-between`, `align-items: center`, `margin-top: 10px`
- «Проверить» слева, «Далее →» справа
- Если кнопка одна — `style="justify-content:flex-end;"` на сам `.nav`

### Шапка страницы
```
.eyebrow  → Courier Prime 12px, letter-spacing 0.12em, uppercase, --pencil
h1        → Caveat 700 40px, line-height 1.05, margin 4px 0 6px
.subtitle → PT Serif italic 15px, --ink-soft, margin 0
header    → margin-bottom 18px
```

## 6. Do's and Don'ts

### Do:
- **Do** использовать только CSS-переменные из `:root` — никаких hex-значений в правилах CSS
- **Do** наклонять только кнопки `.btn` на `rotate(-0.4deg)` — и ничего больше
- **Do** держать `.card` на `rgba(255,255,255,0.55)` — тетрадный фон должен просвечивать
- **Do** использовать Caveat только в `h1` и `.btn`; PT Serif — для всего читаемого текста; Courier Prime — только для лейблов и метаданных
- **Do** ставить `.clip.done` только при `earned[i] > 0` — нулевой балл даёт `.miss`, а не `.done`
- **Do** держать `.stamp` наклонённым на `rotate(-8deg)` — это рукотворная подпись
- **Do** сохранять `max-width: 760px` для `.page` — не 720px, не 800px
- **Do** использовать одинаковые шрифты, палитру и layout-логику на лендинге и в заданиях — родственное, не идентичное

### Don't:
- **Don't** использовать `border-left` шире 1px как цветную декоративную полосу — это абсолютный запрет системы
- **Don't** делать корпоративный ЕГЭ-сайт: синие градиенты, стоковые фото счастливых школьников, таблицы с обещаниями «100 баллов»
- **Don't** добавлять игровые элементы: значки достижений, конфетти, анимированные прогресс-бары, мультяшные иконки
- **Don't** эмулировать советскую школу: тёмно-зелёные тона, тяжёлые рамки, нарочитая архаика
- **Don't** добавлять `box-shadow` на карточки и кнопки — система плоская по принципу, не по случайности
- **Don't** вводить четвёртый шрифт — ни для лендинга, ни для новых заданий
- **Don't** менять CSS-переменные под конкретный файл — если нужно отклонение, зафикси его в задаче
- **Don't** использовать `--red` вне ошибок, активных состояний и номеров вопросов — он работает именно от своей редкости
- **Don't** делать `gradient text` (`background-clip: text` + gradient) — запрещено абсолютно
- **Don't** повторять eyebrow-лейбл над каждой секцией лендинга — один именованный eyebrow как система — голос; eyebrow над каждой секцией — шаблон
